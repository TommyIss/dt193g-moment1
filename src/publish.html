Publicering via Netlify

För publicering har jag valt Netlify som är en modern och anävndarvänlig plattform för att hosta statiska webbplatser. Netlify erbjuder snabb global distribution och automatisk uppdatering från Github, vilket gör det utmärk för mindre projekt och studentuppgifter.

Utvärdering av Netlify - plattform

Funktioner och stöd

Netlify erbjuder ett brett stöd för att hantera statiska webbplatser som är byggda med HTML, CSS, och JavaScript, utan server-logik. Netlify har inbyggda stöd för populära ramverk som Bootstrap och Tailwind CSS. Den erbjuder automatisk bygg- och publiceringsprocess från Github, vilket bidrar till automatisk uppdatering vid kodändring i Github-repot. Dessutom stödjer plattformen även anpassade domäner och omdirigeringar.

Fördelar

Publicering via Netlify har flera fördelar, såsom processen är snabb och enkel från Github-repot, automatisk uppdatering vid varje kodändring, och även stöd för att koppla en egen domän.

Nackdelar

Netlify har vissa begränsningar, såsom den är avsedd för bara statiska webbplatser byggda med HTML, CSS, samt JavaScript och saknar stöd för hosta databaser, API:er eller server-baserade webbtjänster, vilket kan vara ett hinder om man vill bygga mer komplexa applikationer med autentisering eller datalagring.

Publiceringsprocess

Så här gick processen tillväga för att publicera webbplatsen:

  • Publicerade lokala arbetfiler till Github-repo genom att klicka på "Publish to Github"
  • I Netlify-webbsida skapades ett nytt projekt via att klicka på "Add new project", välja "Import an existing project", "Github" och välja avsedda repositoryt.
  • Angav projektnamn "tois-dt193g-moment1", valde branch och i slutet klickade på "Deploy tois-dt193g-moment1"
